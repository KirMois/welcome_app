# Welcome App VLC — Архитектура и структура

## 1. Карта экранов и навигация

### Экраны (Screens)

| ID экрана | Название (RU) | Роль |
|-----------|----------------|------|
| `home` | Домой | Главная: заголовок, подзаголовок, CTA |
| `services` | Услуги / Решения для бизнеса | Список категорий и карточек услуг |
| `solution-picker` | Подобрать решение | Пошаговый мини-квиз + форма контактов |
| `faq` | FAQ / Как мы работаем | Список вопросов и ответов |
| `contacts` | Контакты / Обо мне | Обо мне, кнопки и ссылки |

### Навигация

- **Главное меню (нижняя панель или хедер):** Домой | Услуги | Подобрать решение | FAQ | Контакты  
- **Переходы с Home:**  
  - «Подобрать решение» → `solution-picker`  
  - «Посмотреть услуги» → `services`  
- **Переходы с Services:**  
  - Кнопка на карточке услуги → можно вести на `solution-picker` с предзаполненной категорией или на отдельный экран детали (опционально; в минимальной версии — только `solution-picker`).  
- **Solution Picker:** только линейный поток Step 1 → 2 → 3 → 4 → Submit → экран «Спасибо».  
- **FAQ и Contacts:** доступны из меню, обратно — через меню или кнопку «Назад».

### Поток квиза (Solution Picker)

1. **Step 1:** «Что вам нужно?» — чекбоксы (множественный выбор).  
2. **Step 2:** «На каком вы сейчас этапе?» — один вариант (радио или карточки).  
3. **Step 3:** «Бюджет (ориентировочно)?» — один вариант.  
4. **Step 4:** Контакты: Имя, Telegram @username, «Кратко опишите задачу» + кнопка «Отправить и подобрать решение».  
5. **После отправки:** экран «Спасибо» с текстом про подбор вариантов и связь в течение N часов.

## 2. Список компонентов

| Компонент | Назначение | Где используется |
|-----------|------------|-------------------|
| **AppShell** | Обёртка: хедер, область контента, нижняя навигация | Все экраны |
| **HomeScreen** | Заголовок, подзаголовок, два CTA, вспомогательный текст | Экран `home` |
| **ServiceList** | Контейнер категорий услуг | Экран `services` |
| **ServiceCategory** | Блок категории: заголовок + список карточек + мелкий текст | Внутри ServiceList |
| **ServiceCard** | Карточка: название, описание, цена «от X €», кнопка действия | Внутри ServiceCategory |
| **SolutionPicker (Quiz)** | Пошаговый квиз: шаги 1–4 + финальная форма | Экран `solution-picker` |
| **QuizStep** | Один шаг квиза: заголовок + варианты (чекбоксы/радио) | Внутри SolutionPicker |
| **LeadForm** | Поля: Имя, @username, описание задачи + кнопка отправки | Step 4 квиза; можно переиспользовать |
| **ThankYouScreen** | Сообщение «Спасибо! Я подберу…» после отправки заявки | После submit в solution-picker |
| **FAQList** | Список пар вопрос–ответ (аккордеон или список) | Экран `faq` |
| **FAQItem** | Один вопрос + ответ | Внутри FAQList |
| **ContactScreen** | Текст «Обо мне», кнопка «Написать в Telegram», ссылка на сайт | Экран `contacts` |
| **NavBar** | Нижняя или верхняя навигация по экранам | AppShell |

## 3. Размещение контента по экранам

### Home (Домой)

- **Headline:** «Цифровые решения, которые экономят время и приводят клиентов»
- **Subheadline:** «Создаю ботов, мини‑приложения и веб‑системы под бизнес. От простого приёма заявок до сложной автоматизации и омниканальных решений.»
- **Primary CTA:** «Подобрать решение» → переход на `solution-picker`
- **Secondary CTA:** «Посмотреть услуги» → переход на `services`
- **Helper text:** «Обычно отвечаю в течение 2 часов. Работаю с малым и средним бизнесом в ЕС и СНГ.»

### Services (Услуги)

- Группы (категории) с карточками. В каждой карточке: **title**, **short description (RU)**, **price** «от X €», **action button**.
- Под каждой категорией — **small text** (указан в ТЗ для каждой группы).
- Категории и формулировки — см. раздел «Данные услуг» ниже (в коде в `data.js` или аналоге).

### Solution Picker (Подобрать решение)

- **Step 1:** Заголовок «Что вам нужно?»; чекбоксы с вариантами (тексты из ТЗ).
- **Step 2:** «На каком вы сейчас этапе?»; один выбор из 4 вариантов.
- **Step 3:** «Бюджет (ориентировочно)?»; один выбор из 4 диапазонов.
- **Step 4:** LeadForm: Имя, Telegram @username, «Кратко опишите задачу», кнопка «Отправить и подобрать решение».
- **После submit:** ThankYouScreen: «Спасибо! Я подберу 1–2 оптимальных варианта под ваш запрос и свяжусь с вами в течение N часов.»

### FAQ

- Список вопросов (заголовки FAQItem):
  - «С кем вы работаете: только с малым бизнесом?»
  - «Как быстро вы отвечаете на заявки?»
  - «Можно ли начать с маленького решения и потом масштабировать?»
  - «Делаете ли вы поддержку после запуска?»
  - «Работаете ли вы с клиентами не из Испании?»
  - «Можно ли интегрировать решения с моей текущей CRM/телефонией?»
- Ответы — контент только (структура готова, тексты можно заполнить позже или оставить плейсхолдер).

### Contacts (Обо мне)

- **About text:** «Привет, я [Имя], разработчик решений для бизнеса: боты, мини-приложения и веб‑системы. Помогаю компаниям автоматизировать процессы, не превращая всё в сложный IT‑проект.»
- **Кнопка:** «Написать в Telegram»
- **Ссылка:** https://www.novacodevlc.com (Website)

## 5. Техническая основа

- **Платформа:** Telegram Mini App (WebApp), открывается из бота.
- **Файлы:**  
  - `index.html` — единственная точка входа, контейнер для SPA.  
  - `css/styles.css` — переменные темы (Telegram light/dark), базовая сетка и типографика без визуального полиша.  
  - `js/app.js` — роутер, инициализация Telegram WebApp, рендер экранов.  
  - `js/data.js` — константы: услуги, шаги квиза, FAQ, тексты контактов.
- **Навигация:** hash-based (`#home`, `#services`, `#solution-picker`, `#faq`, `#contacts`) или `history` + один контейнер `#app`.
- **Тема:** CSS-переменные, заполняемые из `Telegram.WebApp.themeParams` / `tgWebAppThemeParams` для светлой и тёмной темы.

Стилевые решения (цвета, шрифты, отступы) не фиксируются в этом документе — только структура, потоки и размещение текста.
