# Welcome App VLC — UX: потоки, формы, данные

## 1. Навигация

**Тип:** нижняя панель (bottom nav), 5 пунктов. Экраны переключаются по hash без перезагрузки.

| Пункт меню | ID экрана   | Описание |
|------------|-------------|----------|
| Домой      | `home`      | Главная: заголовок, подзаголовок, три CTA |
| Услуги     | `services`  | Категории и карточки услуг |
| Подобрать  | `solution-picker` | Мини-квиз из 4 шагов, в конце — те же поля контактов, что и в прямой форме |
| FAQ        | `faq`       | Вопросы и ответы (аккордеон) |
| Контакты   | `contacts`  | Обо мне, ссылки |

**Шестой экран без пункта в меню:** прямая форма заявки `lead-form`. Открывается по кнопке «Оставить заявку» с главной или с карточки услуги. Пользователь возвращается через навигацию (переход на любой из 5 пунктов) или кнопку «Назад» / «На главную».

---

## 2. Потоки

### 2.1 Home → Services → LeadForm (с предвыбранной услугой)

1. Пользователь на **Домой**. Нажимает «Посмотреть услуги» → переход на экран **Услуги**.
2. На экране **Услуги** листает категории, выбирает карточку (например «Standard Bot») и нажимает **«Оставить заявку»** на карточке.
3. Открывается экран **«Оставить заявку»** (lead-form). В поле «Интересует услуга» (опционально) подставлено значение выбранной услуги (например «Standard Bot» или «Боты для бизнеса — Standard Bot»). Остальные поля пустые; Telegram @username по возможности подставляется из Mini App.
4. Пользователь заполняет Имя, при необходимости правит @username, вводит описание задачи, при желании выбирает удобное время для связи. Нажимает **«Отправить заявку»**.
5. Показывается экран **«Спасибо»** с текстом: «Спасибо! Я свяжусь с вами в ближайшее время. Обычно это занимает не больше 2 часов.»
6. Дальше пользователь может перейти «На главную» или выбрать любой пункт в нижнем меню.

**Данные при отправке прямой формы:** см. раздел 4.

---

### 2.2 Home → Quiz (Подобрать решение) → LeadForm (те же поля) → Спасибо

1. Пользователь на **Домой**. Нажимает **«Подобрать решение»** → переход на экран **Подобрать** (квиз).
2. **Шаг 1 из 4:** «Что вам нужно?» — один или несколько вариантов (чекбоксы). Далее «Далее» или автоматический переход.
3. **Шаг 2 из 4:** «На каком вы сейчас этапе?» — один вариант (радио).
4. **Шаг 3 из 4:** «Бюджет (ориентировочно)?» — один вариант (радио).
5. **Шаг 4 из 4:** те же поля, что и в прямой форме заявки: Имя, Telegram @username (можно авто), «Кратко опишите задачу» (2–3 строки, плейсхолдер с примером). Опционально: выбор услуги, удобное время для связи (утро / день / вечер). Кнопка **«Отправить и подобрать решение»**.
6. После отправки — экран **«Спасибо»** с тем же текстом: «Спасибо! Я свяжусь с вами в ближайшее время. Обычно это занимает не больше 2 часов.»
7. Пользователь может уйти «На главную» или в любой раздел через нижнее меню.

**Как данные квиза попадают в заявку:** ответы шагов 1–3 (нужный продукт, этап, бюджет) сохраняются в состоянии квиза и при нажатии «Отправить и подобрать решение» объединяются с полями шага 4 (имя, username, задача, опционально услуга и время) в один payload и отправляются боту через `Telegram.WebApp.sendData()`. Формат — см. раздел 4.

---

### 2.3 Прямая заявка с других экранов

- **Домой:** кнопка **«Оставить заявку»** ведёт на `lead-form` без предвыбранной услуги.
- **Контакты / FAQ:** при необходимости можно добавить кнопку «Оставить заявку» внизу блока, ведущую на `lead-form` (без предзаполнения услуги).

## 3. Поведение форм

### 3.1 Прямая форма заявки (один экран)

- **Экран:** один, без пошаговости.
- **Поля:**  
  - **Имя** — обязательно.  
  - **Telegram @username** — обязательно; по возможности подставляется из Telegram Mini App (initData), пользователь может изменить.  
  - **«Кратко опишите задачу»** — обязательно, 2–3 строки; в placeholder пример: «Например: нужен бот для записи клиентов и уведомлений в Telegram».  
  - **Интересует услуга** — опционально; выпадающий список или выбор из списка услуг; при переходе с карточки — предзаполнено.  
  - **Удобное время для связи** — опционально: утро / день / вечер (радио или селект).
- **Кнопка:** «Отправить заявку».
- **После отправки:** сообщение «Спасибо! Я свяжусь с вами в ближайшее время. Обычно это занимает не больше 2 часов.» и кнопка «На главную».

### 3.2 Квиз «Подобрать решение»

- 4 коротких шага, без перегруза.
- **Прогресс:** над каждым шагом лёгкая подпись «Шаг 1 из 4», «Шаг 2 из 4» и т.д.
- Шаги 1–3: только выбор вариантов. Шаг 4: те же контактные поля, что и в прямой форме (Имя, @username, описание задачи, опционально услуга и время). Одна кнопка в конце: «Отправить и подобрать решение».
- После отправки — тот же экран «Спасибо», что и для прямой формы.

## 4. Данные при отправке

### 4.1 Прямая форма («Оставить заявку»)

Отправляется один JSON-объект через `Telegram.WebApp.sendData(JSON.stringify(payload))`.

| Поле         | Тип     | Описание |
|--------------|--------|----------|
| `source`     | string | Всегда `"lead_form"` |
| `name`       | string | Имя (обязательно) |
| `username`   | string | Telegram @username (обязательно; можно авто-подставить из Mini App) |
| `task`       | string | Краткое описание задачи (обязательно) |
| `service`    | string | Опционально: выбранная услуга (например «Боты для бизнеса — Standard Bot») |
| `contactTime`| string | Опционально: «утро» / «день» / «вечер» |

При переходе с карточки услуги поле «Интересует услуга» предзаполняется, и в payload уходит выбранное значение в `service`.

### 4.2 Квиз («Подобрать решение»)

Отправляется один JSON-объект через `Telegram.WebApp.sendData(JSON.stringify(payload))`. Ответы шагов 1–3 и контакты шага 4 объединяются в один объект.

| Поле         | Тип     | Описание |
|--------------|--------|----------|
| `source`     | string | Всегда `"quiz"` |
| `need`       | string[] | Ответы шага 1: что нужно (массив выбранных строк) |
| `stage`      | string | Ответ шага 2: этап |
| `budget`     | string | Ответ шага 3: бюджет |
| `name`       | string | Имя (шаг 4) |
| `username`   | string | Telegram @username (шаг 4) |
| `task`       | string | Описание задачи (шаг 4) |
| `service`    | string | Опционально (шаг 4) |
| `contactTime`| string | Опционально (шаг 4) |

Бот по полю `source` различает прямую заявку и заявку после квиза и может по-разному обрабатывать (например, для квиза — подбор решения и ответ в N часов).

## 5. Финальные русские тексты

См. файл `js/data.js`: все кнопки, плейсхолдеры, подсказки и ответы FAQ заданы там. Ниже — сводка ключевых формулировок.

### Кнопки и CTA

- Главная: «Подобрать решение», «Посмотреть услуги», «Оставить заявку».
- Карточка услуги: «Оставить заявку».
- Прямая форма: «Отправить заявку».
- Квиз, шаг 4: «Отправить и подобрать решение».
- После отправки: «На главную».

### Форма заявки

- **Имя** (обязательно). Плейсхолдер: «Как к вам обращаться».
- **Telegram @username** (обязательно); подставляется из Telegram, если открыто в Mini App. Плейсхолдер: «например, @username».
- **Кратко опишите задачу** (обязательно, 2–3 строки). Плейсхолдер: «Например: нужен бот для записи клиентов и уведомлений в Telegram».
- **Интересует услуга** — опционально, выпадающий список (все услуги из раздела «Услуги»). При переходе с карточки — предзаполнено.
- **Удобное время для связи** — опционально: Утро / День / Вечер.

### Сообщение после отправки

- «Спасибо! Я свяжусь с вами в ближайшее время. Обычно это занимает не больше 2 часов.»

### FAQ (вопросы и короткие ответы)

Ответы в 2–3 предложения, дружелюбные и практичные; полные тексты заданы в `js/data.js`, раздел `faq.items` (все 6 ответов заполнены).
